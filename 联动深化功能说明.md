# 联动深化功能说明

## 功能概述

本次开发实现了"联动深化"核心功能，将默认建筑和公共点位之间的路线统计、观察文案和季度记录完整串联，使街区人流不再是单向路过，而是能够持续积累成真实的街区习惯。

## 核心功能

### 1. 双向路线统计

**实现原理**：
- 正向统计：公共点位 → 默认建筑
- 反向统计：默认建筑 → 公共点位（通过 `visitor.lastPublicSpotId` 追踪）

**数据结构**：
```javascript
// 每日路线统计（基于建筑实例ID）
state.dailyPublicSpotLinks[key] = {
  id: `${landmark.id}:${publicSpot.id}`,
  landmarkId: landmark.id,
  landmarkType: landmark.type,
  landmarkName: "图书馆",
  publicSpotId: publicSpot.id,
  publicSpotType: publicSpot.type,
  publicSpotName: "口袋广场",
  visits: 5
};

// 季度路线统计（基于建筑类型名称）
state.quarterStats.routineLinks["图书馆 -> 口袋广场"] = 12;
```

**触发时机**：
- 游客访问公共点位时：调用 `notePublicSpotVisit(publicSpot, visitor)`
- 游客访问默认建筑时：调用 `noteLandmarkVisit(landmark, visitor)` 反向统计

### 2. 建筑类型差异化观察文案

#### 路线形成观察（`getRoutineLinkNote`）

**触发时机**：路线访问次数达到 2 次或 4 次时

**差异化文案**：

| 路线组合 | 观察文案示例 |
|---------|-------------|
| 图书馆 ↔ 口袋广场 | "借书的人都会到广场歇一歇" |
| 诊所 ↔ 喷泉角 | "问诊后就到喷泉散散心" |
| 邮局 ↔ 街边小摊 | "寄完顺手买点小吃" |
| 工坊 ↔ 街边小摊 | "下班顺手买点东西回家" |

#### 每日总结观察（`getDailyRoutineSnapshot`）

**触发时机**：每日结束时生成观察记录

**文案示例**：
- "图书馆 -> 口袋广场 这条路线最活跃，学生借完书都去广场歇着，顺便还会逛小吃店。"
- "邮局和街边小摊之间的来往最频繁，寄信的人顺路逛小吃店，办事和买东西成了固定搭配。"

### 3. 季度报表展示

**故事主线**：
- 优先展示热门路线："图书馆 -> 口袋广场 这条来往路线开始稳定下来，街区日常终于接上了。"

**观察趣闻板块**：
- 展示路线累计访问次数："图书馆 -> 口袋广场 本季出现 12 次，默认建筑和公共点位已经开始一起带动街区。"

**指标数据**：
```javascript
{
  topRoutineLink: "图书馆 -> 口袋广场",
  // 其他指标...
}
```

**补看机制**：
- 季节切换时仍会自动弹出季度报表
- 若玩家当时错过，可通过右侧顶部信息卡的 `补看上季` 按钮重新打开最近一次季报
- 最近一季报表会缓存到 `state.lastQuarterReport`，供界面回看和自动化状态输出复用

## 技术实现

### 关键函数

#### `noteLandmarkPublicSpotLink(publicSpot, visitor)`
统计路线访问次数

**参数**：
- `publicSpot`: 公共点位实体
- `visitor`: 游客实体（可选）

**逻辑**：
1. 确定关联的默认建筑（通过 `visitor.errandLandmarkId`、`visitor.originLandmarkId` 或 `publicSpot.linkedLandmarkId`）
2. 更新每日路线统计（`state.dailyPublicSpotLinks`）
3. 更新季度路线统计（`state.quarterStats.routineLinks`）
4. 访问次数达到阈值时触发观察记录

#### `notePublicSpotVisit(publicSpot, visitor)`
记录公共点位访问

**新增功能**：
- 记录 `visitor.lastPublicSpotId`，用于后续反向统计

#### `noteLandmarkVisit(landmark, visitor)`
记录默认建筑访问

**新增功能**：
- 检查 `visitor.lastPublicSpotId`，如果有则反向统计路线
- 清除 `visitor.lastPublicSpotId`，避免错误统计

### 数据流

```
游客访问公共点位
  ↓
notePublicSpotVisit(publicSpot, visitor)
  ├─ 正向统计路线（公共点位 ↔ 默认建筑）
  └─ 记录 visitor.lastPublicSpotId = publicSpot.id

游客访问默认建筑
  ↓
noteLandmarkVisit(landmark, visitor)
  ├─ 检查 visitor.lastPublicSpotId
  ├─ 反向统计路线（默认建筑 ↔ 公共点位）
  └─ 清除 visitor.lastPublicSpotId

每日结束
  ↓
advanceDay()
  ├─ 生成每日总结（getObserverSnapshot）
  ├─ 展示最活跃路线
  └─ 重置 dailyPublicSpotLinks

季节切换
  ↓
buildQuarterReport(stats)
  ├─ 获取季度热门路线（topRoutineLink）
  ├─ 展示在故事主线和观察趣闻
  └─ 积累街区习惯数据
```

## 问题修复

### 1. NPC 停止运动问题

**错误信息**：
```
Uncaught ReferenceError: landmarkType is not defined
    at noteLandmarkPublicSpotLink (game.js:3959:37)
```

**根本原因**：
在 `noteLandmarkPublicSpotLink` 函数中调用 `getRoutineLinkNote()` 时，使用了未定义的变量 `landmarkType` 和 `publicSpotType`。

**修复方案**：
```javascript
// 修复前
const note = getRoutineLinkNote(landmarkType, publicSpotType, ...);

// 修复后
const note = getRoutineLinkNote(landmark.type, publicSpot.type, ...);
```

### 2. 主循环条件优化

**原问题**：
```javascript
if (!automationDriven && now >= manualControlUntil) {
  update(delta);
}
```

可能导致误判，在自动化环境中游戏冻结。

**修复后**：
```javascript
if (!automationDriven || now >= manualControlUntil) {
  update(delta);
}
```

确保正常浏览器环境下始终更新游戏状态。

### 3. 字符串空值保护

**优化点**：
```javascript
// 添加空值保护
const leftName = left.publicSpotName || "";
const rightName = right.publicSpotName || "";
return leftName.localeCompare(rightName, "zh-Hans-CN");
```

## 使用示例

### 正常游戏流程

1. 放置默认建筑（图书馆、诊所等）和公共点位（口袋广场、喷泉角等）
2. 游客访问公共点位后，继续访问默认建筑
3. 系统自动统计路线访问次数
4. 访问次数达到 2 次时，触发观察记录
5. 每日结束显示最活跃路线
6. 季节切换时，季报展示热门路线和街区习惯形成过程

### 调试测试

3. 观察自动生成的调试路线
4. 验证路线统计和观察文案效果
